#### 发现3：内容沉淀是高价值用户的核心需求

**左侧卡片内容**（橙色渐变背景）：
```
📦 内容沉淀是高价值用户的核心需求

竞品沉淀机制：
• 有道词典：生词本（自动同步）+ 单词笔记（识别圈画）
• 作业帮：错题本 + 学习报告
• 所有竞品：历史记录 + 收藏夹

用户价值：
• 用得越多，沉淀的内容越有价值
• 切换成本越高，留存越稳定
• 形成"使用 → 沉淀 → 复习 → 再使用"的闭环

数据支持：
• 有道用户评论："生词本功能是我离不开的原因"
• 作业帮用户评论："错题本帮我提分，舍不得换"
• 竞品对比：所有6款竞品都有历史记录，5款有收藏功能

小猿现状：
• ❌ 无历史记录（最基础功能）
• ❌ 无收藏/笔记功能
• ❌ 翻译完就消失，无法回看

💡 结论：历史记录是基础需求，收藏/笔记是留存关键
```

- 气泡大小：功能复杂度
- 显示"内容沉淀能力"与"留存"的正相关
- 小猿位于左下角（沉淀能力0分）

```
⚡ 框选翻译和多页扫描是效率刚需

• 场景1：只想翻译部分内容（如：只翻译题目，不翻译选项）

  → 需要多页扫描

• 框选翻译：
  - 夸克扫描王：✅ 手指涂抹选区

  - 其他竞品：❌ 无
• 多页扫描：
  - 夸克扫描王：✅ 一次拍多页，统一处理
  - 其他竞品：❌ 逐页拍摄

用户反馈验证：
• 定量数据：日均翻译2-3张（连续拍摄场景）

• ❌ 无框选翻译（只能整页翻译）
• ❌ 无多页扫描（需要重复"再拍一张"）
• ✅ 有"再拍一张"功能（但体验不够流畅）
```

  - 场景1：只想翻译部分内容（用户占比估计）
  - 场景2：连续拍多张（基于日均2-3张数据）

#### 3.2 优先级1：基础体验优化（P0 - 必须做）

**卡片样式**：红色边框 + 红色标题

```
🔴 优先级1：基础体验优化（P0 - 必须做）

1️⃣ 历史记录功能 ⭐⭐⭐⭐⭐
   【为什么是P0】
   • 所有6款竞品都有（基础功能）
   • 用户反馈："翻译完找不到了"
   • 低频场景（7天用1次）更需要历史记录
   
   【功能设计】
   • 自动保存翻译历史（最近30天）
   • 支持按时间/语言筛选
   • 点击可重新打开翻译结果
   
   【预期效果】
   • 满足基础需求，补齐功能短板
   • 提升用户满意度
   • 为内容沉淀打基础

2️⃣ 多页扫描优化（改进"再拍一张"） ⭐⭐⭐⭐
   【为什么是P0】
   • 用户日均翻译2-3张（连续拍摄场景）
   • 当前"再拍一张"体验不够流畅
   • 夸克扫描王的核心优势
   
   【功能设计】
   • 拍照后直接提示"继续拍摄"或"完成"
   • 批量处理多张图片
   • 统一展示所有翻译结果
   
   【预期效果】
   • 提升批量翻译效率
   • 减少重复操作
   • 适配练习册场景

3️⃣ 框选翻译 ⭐⭐⭐⭐
   【为什么是P0】
   • 灵活性需求（只翻译部分内容）
   • 夸克/百度独有功能
   • 差异化竞争点
   
   【功能设计】
   • 拍照后支持手指涂抹选择区域
   • 只翻译选中部分
   • 参考夸克扫描王的交互
   
   【预期效果】
   • 提升使用灵活性
   • 满足部分翻译需求
   • 差异化竞争

【实现周期】：4周
【资源需求】：前端2人 + 后端1人
```

```
   • 成分标注（主谓宾）

   • 需要接入语法分析模型
   • 或与小猿AI能力整合
```
